## 1. État du projet

- Application **ChefSuite** = gestion de restaurant (React 19 + Express + Socket.io + PostgreSQL + Drizzle + Firebase Auth).
- Front **et** back tournent ensemble sur **http://localhost:3000** via `npm run dev`.
- L'app est **fonctionnelle** en local (base créée, Firebase configuré, comptes réels opérationnels).

### Démarrer
```powershell
npm run dev        # lance le serveur (front + back) sur http://localhost:3000
npm run db:push    # (re)crée/synchronise les tables depuis le schéma Drizzle
npm run lint       # type-check TypeScript (doit finir sans erreur)
```

## 2. Configuration locale (rappel)

- **PostgreSQL** : base `chefsuite`. Accès dans `.env` (`SQL_HOST`, `SQL_DB_NAME`, `SQL_USER`, `SQL_PASSWORD`, `SQL_ADMIN_*`).
  - PostgreSQL 16 installé sous `C:\Program Files\PostgreSQL\16\bin`.
- **`.env`** à la racine : contient aussi `ENABLE_TEST_LOGIN="true"` (bouton d'accès test) et `GEMINI_API_KEY` (optionnel).
- **Firebase** : projet **`chefsuite-local`**, config dans `firebase-applet-config.json`. Auth **Email/Mot de passe activée**. Le back n'utilise que le `projectId` (pas de clé privée), mais a besoin d'Internet.
- **Login de test** : bouton « Accès test (Admin) » → entre sans Firebase (super_admin). Gated par `ENABLE_TEST_LOGIN`.
- ⚠️ Gotcha : la config Drizzle est dans `src/db/drizzle.config.ts` (pas à la racine) → d'où le script `db:push`.
- ⚠️ Gotcha : `localhost` résout parfois en IPv6 ; le serveur écoute en IPv4 → tester avec `127.0.0.1` en ligne de commande.

- **Catégories enrichies** + **personnalisables** (entrée chaude/froide, snack, boisson chaude/fraîche/alcoolisée, cocktail… + « Autre » libre). Filtres dynamiques.

### Personnel
- **Rôles métier** : ajout de *Chef cuisinier* et *Femme de ménage* (cuisinier séparé).

### Multi-utilisateur (super admin ↔ gérant)
